概述

此文档列举了前后端通讯时的几种报文格式。

请求URL格式

请求URL类似于：http[s]://IP:Port/a/d/c，其中http[s]://IP:Port是相对固定的，可做常量配置。

报文基本格式

请求包括
  1. 请求方法：
    • GET 一般用户读数据
    • POST/PUT 一般用于创建或者更新数据
    • DELETE 一般用于删除数据
  2. 请求头，即Header部分：
    • X-AUTH-TOKEN：用于传递请求token
    • Content-Type：见报文
    • Accept：见报文
  3. 请求体
    • form格式或者json格式
响应
  1. HTTP状态码，见码表
    • 做响应处理时，应该首先判断HTTP状态码
  2. 响应头
    • 处理响应体时，应该首先确定Content-Type
  3. 响应体
    • 根据响应头中的Content-Type处理
    • 注意：业务异常时，有个bizcode可以做业务处理
    • 注意：业务异常或者系统异常时，有个ddzcode标记出问题的系统

报文

正常报文

正常的请求，返回状态码一般为2xx，可能不仅仅是200；
正常的请求，前端直接处理返回数据即可。

请求：
curl --include --location --request GET 'http://127.0.0.1:9000/api/test/link/get'
或者：
curl --include --location --request GET 'http://127.0.0.1:9000/api/test/link/get?test1=123123'
响应：
HTTP/1.1 200 OK
transfer-encoding: chunked
Content-Type: application/json;charset=UTF-8
Date: Sat, 15 Feb 2020 21:25:16 GMT

{"key_a1":"中文val"}

请求：
curl --include --location --request POST 'http://127.0.0.1:9000/api/test/link/post' \
--header 'Content-Type: application/json; charset=utf-8' \
--data-raw '{
    "test1": "中文",
    "test2": "23a324"
}'
响应：
HTTP/1.1 200 OK
transfer-encoding: chunked
Content-Type: application/json;charset=UTF-8
Date: Sat, 15 Feb 2020 21:27:57 GMT

{"key_a1":"中文val"}

请求：
curl --include --location --request POST 'http://127.0.0.1:9000/api/test/link/post' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'test1=中文' \
--data-urlencode 'test2=123'
响应：
HTTP/1.1 200 OK
transfer-encoding: chunked
Content-Type: application/json;charset=UTF-8
Date: Sat, 15 Feb 2020 21:40:33 GMT

{"key_a1":"中文val"}

注意：Content-Type: application/json; charset=utf-8或者Content-Type: application/x-www-form-urlencoded会对服务器取值造成影响！使用时需和后端商议妥当。

业务异常报文

业务异常代表程序在处理业务时，遇到条件不符合的情况，而终止了请求，比如：用户年龄大于xx、身份证格式不符合等；
注意bizcode，后端可以设置bizcode和前端通讯，前端可以利用这个bizcode做一些判断操作；
业务异常一般都是4xx的状态码；
业务异常一般直接判断提示message字段返回的内容即可。

请求：
curl --include --location --request POST 'http://127.0.0.1:9000/api/test/link/post' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-raw '{
    "test1": "中文",
    "test2": "23a324"
}'
响应：
HTTP/1.1 402 Payment Required
Content-Type: application/json;charset=UTF-8
Content-Length: 163

{"timestamp":"2020-02-16T20:36:32.818+0000","path":"/api/test/link/get","status":402,"error":"Payment Required","message":"错误测试","bizcode":221,"ddzcode":0}

系统代码异常报文

系统代码异常表示后端系统在编码上出现了错误，这种情况应该友好的提示前端用户。

请求：
curl --include --location --request GET 'http://127.0.0.1:9000/api/test/link/get?test1=123123'
响应：
HTTP/1.1 500 Internal Server Error
Content-Type: application/json;charset=UTF-8
Content-Length: 156

{"timestamp":"2020-02-16T20:34:59.857+0000","path":"/api/test/link/get","status":500,"error":"Internal Server Error","message":null,"bizcode":0,"ddzcode":0}

注意：
不管是业务异常还是系统异常，皆有一个ddzcode错误码，此错误码代表发生问题的系统，比如，系统后台调用了核心系统，当因为核心系统的问题，发生错误时，此ddzcode会返回代表核心系统的码值，方便前端提示给用户到底是哪个系统出现了问题；具体码值，前后端开发时商议而定。

其他报文

Accept目前只支持application/json
请求：（注意Accept设置后的响应！）
curl --include --location --request POST 'http://127.0.0.1:9000/api/test/link/post' \
--header 'Content-Type: application/json; charset=utf-8' \
--header 'Accept: text/plain' \
--data-raw '{
    "test1": "中文",
    "test2": "23a324"
}'
响应：
HTTP/1.1 406 Not Acceptable
Content-Length: 0
Date: Sat, 15 Feb 2020 21:33:43 GMT

请求：
curl --include --location --request POST 'http://127.0.0.1:9000/api/test/link/post' \
> --header 'Content-Type: application/json; charset=utf-8' \
> --header 'Accept: application/json; charset=utf-8' \
> --data-raw '{
> "test1": "中文",
> "test2": "23a324"
> }'
响应：
HTTP/1.1 200 OK
transfer-encoding: chunked
Content-Type: application/json;charset=utf-8
Date: Sat, 15 Feb 2020 21:35:12 GMT

{"key_a1":"中文val"}
